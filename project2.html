Project 2 - Selenium Unit Testing Framework

Idea

Testing is all about automation right? You are developing a new project and there comes the time when you have to check if everything that has been done so far is running smoothly.

If you are not building a very complex project you can perform those tests manually. But that can get very tedious very fast. So what can be done about it to make it less of a hassle? Enter Selenium.

Approach

I have decided to create a small framework that may be applied and adjusted to your liking depending on what it is that you are trying to test.

Step 1

We need to create a few files in which we will store all of our usability. All of those files will be building blocks for our main file. So first, let us start with a page object.

Each page on our website we want to define inside of a class in order to easily check for everything we want. This is where the most of Selenium code is written.

We set up a Base Page class which is the base class for all of our pages. After that we create our MainPage (which essentially is a home page) and LoginPage (which is a page after we log in). We have access to the constructor method (or any other methods we define inside BasePage) because every page class we define inherits from the BasePage. Each class has access to the locators that are specified in the next step. In order to click a login button we want to find element that is denoted by locator that is put in the MainPageLocators in the locators.py in the next step. Adding an asterisk lets us unpack the tuple and pass it as an argument.

Step 2

Lets take a look at locators that we have to get access to.

Any way we locate elements by is kept here in one organized file. It makes changing id's or css selectors easy. Notice capitalized letters as we define our locators as constants.

We create classes that represents objects that we want to find, so in our case we need access to the MainPage locators and LoginPage locators.

Step 3

Lets take a look at element object.

In this file we basically want to represent a single element on the page. We define our set and get methods and what we are saying here is that when we have an element on our page and we want to set or get the value we follow the process inside our method. As for the get method what that says is that every time we set up that base object and any time we want to access or change value it will automatically implement that functionality. In our example that is applicable to the search bar. So we locate that element, we clear the input field and send whatever value is passed.

Step 4

Finally we arrive at our main file, which combines everything and this is where we write our tests.

Here we create all of our classes for test cases that we want to perform on our website. In this example we define class Login as our test case. So our test looks like this:

  • Check if the title of the page has the word Orange in it
  • Locate the input field for login and pass the value
  • Locate the input field for password and pass the value
  • Click login
  • Click the dropdown menu
  • Click logout

And we are done!
